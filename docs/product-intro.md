# Chord 回响 · 当收藏开始回应你

> 一个 Chrome 扩展，把"数字囤积"变成"和自己的对话"。
>
> 念念不忘，必有回响。

---

## 一、开篇 · 你的收藏夹欠你一个故事

打开你的 Chrome 书签栏。

数一数有多少条。

再问自己一个问题：**你能说出最近三个月真正打开过、读过、用过的，有几条？**

研究人员把这种行为叫做"信息囤积"——我们以为收藏即拥有，把"将来会看"的承诺塞进书签栏，然后再也不打开。

数字时代每个人都欠自己一笔阅读债。它不催你，但它在心里。

我们见过的真实数字：
- 一个工程师，**170 条收藏，处理率 6%**
- 一个产品经理，**900 条收藏，30 天新增 42 条，回访 6 条**
- 一个研究生，**3,000 条 arXiv 论文 + 教程，自评"焦虑"**

这些人都不是懒——他们都是对世界保持好奇的人。但他们的工具系统辜负了他们。**Notion 帮你存，Pocket 帮你存，浏览器帮你存——但没有谁帮你回头**。

Chord 想做那个回头的人。

## 二、Chord 是什么

> **Chord 不是书签管理器。是内容与自我的对话工具。**

打开扩展，你会看到的不是一个列表，而是**一条从书房里浮起来的内容**——

- 来源、保存时间、AI 给你写的一句私人问候
- 三个选择（其实是两个，下面会讲为什么）
- 留下来，或者放手

每天一次，慢慢地，温柔地，让那些被搁置的内容回到你眼前。

不是审判，不是清单——是**回响**。一个你曾发出的念头，现在被某个东西轻轻回应了你。

### 三个核心体验

#### 1. 每日回响（Popup）
打开扩展 → 一条收藏静静浮出 → 你和它对视一会儿 → 留下，或放手。
没有 KPI，没有清单焦虑。只有"今天，跟这条说一声话"。

#### 2. 候响室（书房总览）
左侧脉冲呼吸：「170 条等你」+ "都还在，不急" 副标
右侧按主题分组的待响列表，等待时长用色温区分
不催你处理，但让你随时知道"我的内心都在等什么"

#### 3. 隐性自我（你不知道你是谁）
AI 看着你的 90 天行为，告诉你 3 件你自己没意识到的事：
> 「你保存的速度远超处理的速度。170 条里 94% 还没打开。」
>
> 「但有件事让我意外——你在『个人创作与生活』上只有 3 条收藏，却在 90 天里回访了 34 次。这是行动在说话。」
>
> 「另一面是焦虑：4 个主题积压了 145 条，你处理过 5 条。"

这种被看见的感觉——一个工具真的认识你——是 Chord 最核心的体验。

## 三、设计哲学：用户的心，系统的脑

大部分知识管理工具让用户做大量分类决策——这是 tag、那是 folder、这是 archive、那是 starred。**用户被工具雇佣，成了自己内容的产品经理**。

Chord 反过来。

### 「用户只回答情感问题，系统替你思考」

我们做了一个看似激进的简化——**所有决策只剩两个选择**：

- 📌 **留下来** — 我还需要它
- 🌸 **放手** — 我不需要了

「用过了 / 待办 / 长期参考 / 暂存」这种分类，**全部交给系统从行为信号里自己看出来**（你访问几次、留没留笔记、什么时候保存的、跟其他主题的关系）。

用户只回答"还要不要"——一个情感问题。

放手时多一步：**为什么放手？**（已经用过了 / 不感兴趣了 / 当时存错了 / 找到更好的了 / 没时间看了 / 自己说）。这个原因不只是数据——是一次温柔的反思机会，"我跟它的故事到这里了"。

## 四、「念念不忘，必有回响」如何变成功能

这是 Chord 的灵魂。**不只是 slogan**——是产品骨架。

```
用户的「念」（行为信号）→ 系统的「响」（主动回响）
      ↑                                ↓
      └────── 用户的回应（决策 / 对话）──────┘
```

具体地：
- 一条收藏，你 7 天内点开看了 5 次但没决策 → 系统温柔问一句：「这条你回访了 5 次——它对你意味着什么？」
- 你说"留下来"承诺要用 → 60 天后系统会回响：「60 天前你立约说要用它整理读书笔记，做了吗？」
- 你脑子里浮现关键词「专注力」→ 倾听模式让你直接召唤书房中相关的所有内容
- 兴趣地形的气泡，**回响越强水波越大**——视觉化你"内心的回响"

每一个功能都在回答同一个问题：**这个工具能不能听见你的内心？**

## 五、最有意思的部分：12 个身份，3 张牌

当 AI 看了你 90 天的行为之后，它会给你**三张身份卡**叠在一起：

| 维度 | 称号举例 | 一句话 |
|---|---|---|
| **消费方式** | HOARDER 收藏家 / CURATOR 策展人 / EXECUTOR 实践者 / THINKER 思考者 / SLOW READER 慢读者 | 你怎么对待收藏？ |
| **当下心境** | EXPLORER 探索者 / SEEKER 求索者 / RETURNER 回归者 / SETTLER 沉淀者 | 你最近处在什么状态？ |
| **关注半径** | SPECIALIST 专精派 / GENERALIST 广博派 / SWITCHER 跳跃者 | 你的注意力分布有多宽？ |

**5 × 4 × 3 = 60 种组合**，足够区分每个人。

每张牌**正面是一个超大的 serif 字母 + 中文称号**，背面是数据证据。你可以翻牌、收藏、**导出 PNG 分享朋友圈**——像 16Personalities 那种"测出我是 INTJ"的传播感，但 Chord 的版本是**基于真实行为而不是问卷自报**。

视觉上每个身份会配一张**手绘水彩人物形象**（宫崎骏风格）：
- HOARDER：被珍藏物温柔包围的人，富足而非焦虑
- SEEKER：钻入秘境深处、手持火把、紧盯目标
- SPECIALIST：在专属工坊里被工具墙环绕的匠人
- ……（共 12 张，已完成 prompt 设计，待 GPT Image / 即梦出图）

## 六、一个产品经理 + AI 副驾 = 一个完整的 Chrome 扩展

这是 Chord 最让人不可思议的一面：**它由一个产品经理用 AI 完整构建**。

### 时间线（真实，可考据）

**项目起点**：一个想法 + Claude Code（Anthropic 出品的 AI 编程助手）

**第 1 周**：用 AI 把产品需求转成完整 Chrome 扩展骨架——Preact + Chrome MV3 + chrome.storage + IndexedDB + AI 引擎抽象层。所有的"哪个文件该放哪"，AI 决定。

**第 2-4 周**：核心功能。每日回响、候响室、兴趣地形、隐性自我、聚类算法、URL 智能分类、6 套皮肤、Onboarding、数据导出……一个一个长出来。

**第 5-7 周**：聚类算法的痛苦优化。
- 用户截图反馈："『纹藏 · agi』装了 100 条无关内容"
- 一轮一轮调：TF-IDF → AI 聚类 → max_tokens 截断 bug → L1 N 选 1 分类 → prompt 优化 → bad case 库
- 最终：真实数据 168 条达成 **78.9% 准确率**（行业 baseline 是 20%），并建立持续评测体系防回归

**第 8-10 周**：洞察模型升级。AI 头条、focal Findings、人格视图、念念回响功能设计、身份系统传播方案。

**第 11 周**：二向决策升级。把三向决策简化为二向（留下来 / 放手），用户心智极简化，系统通过行为信号自动判断。完整迁移老数据，212 个测试全过。

**至今**：约 ~12 周。**0 个工程师全职。0 行代码"独立"写出**——所有代码出自 AI 之手，但**所有产品决策、所有体验细节、所有"对不对"的判断**，由产品经理把关。

### AI 不是工程师的替代品——是创造力的放大器

这个项目验证了一件事：当一个清晰的产品愿景遇到能动手的 AI，**做产品的速度可以快 5-10 倍**。但 AI 没有判断力——它需要被引导：
- 「这种 prompt 硬规则是错的，会引入新偏见」 → AI 删了硬规则、换成语义意图描述
- 「分类还是不准」 → AI 反复诊断 → 找到 max_tokens=512 硬编码导致 AI 输出被截断的 root cause
- 「用户心智模型有问题，留下来 vs 用过了边界模糊」 → AI 提出二向决策方案 → 一起设计完整迁移路径

**AI 跑代码 + 跑测试 + 写文档 + 推断 bug，人做"对不对"的判断**。这是新一代产品研发的模式。

## 九、不一样的地方

| 维度 | Notion / Pocket / Raindrop | Chord |
|---|---|---|
| 关注点 | 收藏与管理 | 收藏与自我对话 |
| 用户决策点 | 多（tags / folders / lists） | 二选（留 / 放） |
| AI 角色 | 搜索增强 | 洞察 + 回响 + 行为分析 |
| 数据隐私 | 必须上云 | 默认本地，云端可选 |
| 情感设计 | 工具感 | 仪式感 + 诗意感 |
| 视觉哲学 | 信息密度 | 留白 + 浮云 + 樱花 |

Chord 不跟它们竞争——Chord 是不同物种。
